How this works

USPS Informed Visibility — Mail Tracking & Reporting (IV-MTR) schedules a Data Delivery subscription that POSTs JSON to a URL we own. USPS' source IPs are not fixed/published, so this endpoint authenticates via HTTP Basic Auth. TLS is terminated by your front-end proxy (Caddy / Cloudflare / nginx).

Your endpoint URL (paste into the IV portal as Host Address + Host Target Directory):

https://<your-public-host>/usps_feed

USPS sends file in the Subscription's configured frequency (e.g. every 1 hour). Heads up: if a delivery returns 5xx, USPS retries on its next cycle — duplicate scans are deduplicated server-side by (scan_date_time, event_code, machine_name, facility_zip, facility_locale).

USPS portal — what to fill in

In the IV-MTR portal, create a Subscription → Delivery tab:

Protocol TypeHTTPS JSON
Host Addressyour public host (no scheme), e.g. track.example.com
Port443
Host Target Directory/usps_feed
Host User Namethe Basic Auth username from above
Host Passwordthe Basic Auth password from above (use Rotate to generate)
File FormatJSON
File Transfer FormatUn-zipped for MVP, or Zipped if you flip Expect gzip on
Frequencye.g. 1 Hour (depends on your IV agreement)

Field selection: on the "Define Filters & Fields" tab make sure you include at least imb, scanDateTime, scanEventCode, handlingEventType, and the facility fields (scanFacilityName, scanFacilityCity, scanFacilityState, scanFacilityZIP). Missing fields aren't fatal — the parser only requires imb.
