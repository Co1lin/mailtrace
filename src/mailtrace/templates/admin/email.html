Provider field guides

Copy-paste-friendly settings for the common cases. The plaintext SMTP password is stored in the local SQLite database — for shared SMTP services, use a scoped API key rather than your real password whenever possible.

Office 365 / Microsoft 365 (work or school account)

Hostsmtp.office365.com
Port587
EncryptionSTARTTLS
Usernameyour full email address (e.g. you@yourorg.onmicrosoft.com or you@yourcompany.com)
Passwordyour account password, or an app password if MFA is on
From addresssame as username (Microsoft rejects mismatched From)

Heads up — common gotchas:

  • Your tenant admin must enable Authenticated SMTP for your mailbox. In Microsoft 365 admin center → Users → pick the user → Mail → Manage email apps → make sure Authenticated SMTP is on.
  • If MFA is enabled on the account, your normal password will not work over SMTP. Generate an app password at https://account.microsoft.com/security → Advanced security options → App passwords.
  • Microsoft has been disabling basic SMTP auth across many tenants. If your tenant has it locked off and you cannot enable it, use a relay (SendGrid / Postmark / Resend / AWS SES) instead.

Gmail / Google Workspace

Hostsmtp.gmail.com
Port587
EncryptionSTARTTLS
Usernameyour full Gmail / Workspace address
Passworda 16-character app password (not your normal password)

Generate an app password at https://myaccount.google.com/apppasswords. 2-step verification on the account is required to create one. App passwords show up like abcd efgh ijkl mnop — paste them with or without spaces, both work.

Transactional relay (SendGrid, Resend, Postmark, Mailgun, AWS SES)

Sign up for any of these — they all have free tiers that cover homelab volume — and use the SMTP credentials they hand you. The pattern is always:

Hoste.g. smtp.sendgrid.net / smtp.resend.com / smtp.postmarkapp.com
Port587 (STARTTLS) or 465 (TLS)
Usernameliteral string the provider tells you to use (often apikey)
Passwordthe API key the provider gave you

Why pick this: better deliverability than home-IP / consumer SMTP, no MFA / app-password dance, dedicated webhooks for bounces, and revocable per-app keys.
