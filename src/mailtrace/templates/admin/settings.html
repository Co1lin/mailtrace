What lives here

Platform-wide operational tuning that applies to every tenant. USPS API credentials are NOT here — they are per-user, configured by each user on their own Account page. As the platform admin you only manage users (/admin/), SMTP (/admin/email), the IV-MTR push receiver (/admin/ingest), and the background poller cadence below.

Background poller cadence

The background loop wakes every interval seconds, polls in-flight pieces (each using their owner's BCG credentials), runs auto-archive, and dispatches email digests. Changes take effect on the next loop tick — no restart needed.

Where else config lives

  • Users — invite, force-reset, deactivate, delete.
  • SMTP server — for the email digest the poller / push-feed sends to opted-in users.
  • USPS push feed — Basic Auth + gzip + raw-payload archive for the IV-MTR push receiver. One platform-level Basic Auth credential; each user creates their own USPS-side subscription pointing at the same endpoint.
  • Per-user: each user opens Account to set their own Mailer ID, USPS API Client ID/Secret, and (optionally) BCG credentials.
  • Bootstrap (env vars only): MAILTRACE_SESSION_SECRET, MAILTRACE_DATABASE_URL, MAILTRACE_REDIS_URL, plus host/port and infrastructure paths.
