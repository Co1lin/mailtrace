1 · Mailer ID (required)

Your USPS-issued Mailer ID, found in your Business Customer Gateway account at gateway.usps.com under Mailing Services → Mailer ID. Required to create any mail piece. Two users can never share a MID — that would collide on USPS's (MID, serial) keyspace.

Advanced: Barcode ID and Service Type ID — leave at the defaults unless you know you need to change them

Barcode ID (BI) is a 2-digit code that groups a sender's mailings on the USPS side. For personal / single-mailer use the default 0 is correct — change this only if USPS issued you multiple BIs and you want to tag pieces against a specific one (e.g. to separate billing).

Service Type ID (STID) is a 3-digit code that tells USPS what mail class + tracking services the piece is for. The default 40 is "First-Class Mail, IMb tracking with IV-MTR" — the right value for postcards / letters dropped in a regular mailbox. Other classes (Marketing Mail, Periodicals, certified, return-receipt, …) have different STIDs from USPS's STID table; don't guess — look up the value for the exact service you're paying for, or your mail can be routed wrong.

For homelab / occasional personal use: just leave both at the defaults. MailTrace was designed around BI=0 / STID=40 and that's what the upstream 1997cui/envelope deployment runs as well.

2 · USPS API credentials (optional, recommended)

For the in-page Validate against USPS button on piece / address forms. Without these, the button returns an error but you can still type addresses by hand.

How to get them
  1. You need a BCG account first — gateway.usps.com, sign up for a Business Account.
  2. Go to developer.usps.com and click Get Started; follow the prompts. You'll end up on the USPS Customer Onboarding Portal at cop.usps.com, signed in with your BCG account.
  3. In cop.usps.com look for the small My Apps button in the top-right corner. Click it and create a new app (any name — e.g. "MailTrace").
  4. Once the app is created, its detail page shows a Consumer Key and Consumer Secret. Copy both.
  5. Paste the Consumer Key into Client ID below and the Consumer Secret into Client Secret, then Save all sections at the bottom of this page.
  6. Click Test USPS API in the "Test your setup" section to verify. (USPS' edge can take ~10 min to propagate after app creation; if the first test fails with a 401, wait a bit and retry.)

USPS' onboarding flow has changed before and may change again — these are the URLs and screens as of writing. If you land somewhere different, the goal is unchanged: end up with a Consumer Key + Consumer Secret pair from a USPS developer app, then paste them below.

3 · BCG credentials (optional, only if you want pull-tracking)

Used by the platform's background poller to fetch scan history from USPS' legacy IV-MTR pull API on your behalf. Skip this section if your scans come via the platform's push receiver — the modern alternative requires no per-user USPS credentials.

USPS' legacy IV-MTR pull API only accepts OAuth2 password grant — there is no API-key equivalent. These are your literal username and password from gateway.usps.com. They're stored in the platform DB; rotate by changing your BCG password and updating this field.
