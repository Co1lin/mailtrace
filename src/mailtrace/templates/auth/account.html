1 · Mailer ID (required)

Your USPS-issued Mailer ID, found in your Business Customer Gateway account at gateway.usps.com under Mailing Services → Mailer ID. Required to create any mail piece. Two users can never share a MID — that would collide on USPS's (MID, serial) keyspace.

2 · USPS API credentials (optional, recommended)

For the in-page Validate against USPS button on piece / address forms. Without these, the button returns an error but you can still type addresses by hand.

How to get them (5 min)
  1. You need a BCG account first — gateway.usps.com, sign up for a Business Account.
  2. Go to developer.usps.com and sign in with your BCG account.
  3. Add App → name it "mailtrace" → enable Addresses 3.0.
  4. Save. The app's detail page shows a Consumer Key and Consumer Secret.
  5. Paste them below. (USPS' edge can take ~10 min to propagate; if a Test fails right after creating the app, wait and retry.)

3 · BCG credentials (optional, only if you want pull-tracking)

Used by the platform's background poller to fetch scan history from USPS' legacy IV-MTR pull API on your behalf. Skip this section if your scans come via the platform's push receiver — the modern alternative requires no per-user USPS credentials.

USPS' legacy IV-MTR pull API only accepts OAuth2 password grant — there is no API-key equivalent. These are your literal username and password from gateway.usps.com. They're stored in the platform DB; rotate by changing your BCG password and updating this field.
